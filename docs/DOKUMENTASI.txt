1. PENJELASAN SINGKAT STRUKTUR APLIKASI
================================================================================

SIJAtools adalah aplikasi desktop ringan untuk manajemen peminjaman alat. 
Aplikasi ini dibangun dengan teknologi:
  
  ✓ Bahasa: Python
  ✓ UI Framework: Tkinter (built-in Python)
  ✓ Database: SQLite (file lokal: sijatools.db)
  ✓ Koneksi: Offline, tidak memerlukan internet

ARSITEKTUR:
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  ┌──────────────────┐              ┌──────────────────────┐               │
│  │  GUI Layer       │              │  Database Layer      │               │
│  │ (Tkinter UI)     │──────────────│ (SQLite)             │               │
│  │                  │              │                      │               │
│  │ - 4 Tab Menu     │     Call     │ - Tabel alat         │               │
│  │ - Form Input     │────────────→ │ - Tabel peminjaman   │               │
│  │ - Tabel Treeview │   Query      │ - CRUD Operations    │               │

2. STRUKTUR DATABASE
================================================================================

DATABASE FILE: sijatools.db
Lokasi: Folder yang sama dengan file SIJAtools.py

DATABASE SCHEMA:
──────────────────────────────────────────────────────────────────────────────

TABEL 1: alat
┌──────────────┬────────────────┬──────────────────────────────────────┐
│ Column       │ Type           │ Deskripsi                            │
├──────────────┼────────────────┼──────────────────────────────────────┤
│ id           │ INTEGER (PK)   │ ID unik, auto-increment              │
│ nama_alat    │ TEXT (UNIQUE)  │ Nama alat (tidak boleh duplikat)     │
│ stok         │ INTEGER        │ Jumlah stok tersedia                 │
│ deskripsi    │ TEXT           │ Deskripsi tambahan alat              │
│ tanggal_[+]  │ TIMESTAMP      │ Waktu alat ditambahkan               │
└──────────────┴────────────────┴──────────────────────────────────────┘

TABEL 2: peminjaman
┌──────────────────────┬────────────────┬──────────────────────────────────┐
│ Column               │ Type           │ Deskripsi                        │
├──────────────────────┼────────────────┼──────────────────────────────────┤
│ id                   │ INTEGER (PK)   │ ID unik peminjaman               │
│ nama_peminjam        │ TEXT           │ Nama orang yang meminjam         │
│ id_alat              │ INTEGER (FK)   │ Reference ke tabel alat          │
│ jumlah               │ INTEGER        │ Jumlah alat yang dipinjam        │
│ status               │ TEXT           │ 'Dipinjam' atau 'Dikembalikan'   │
│ tanggal_peminjaman   │ TIMESTAMP      │ Waktu peminjaman                 │
│ tanggal_pengembalian │ TIMESTAMP      │ Waktu pengembalian (null jika    │
│                      │                │ belum dikembalikan)              │
└──────────────────────┴────────────────┴──────────────────────────────────┘

RELASI:
peminjaman.id_alat → alat.id (Foreign Key)

================================================================================
4. PENJELASAN BAGIAN PENTING KODE
================================================================================

A. CLASS DatabaseManager
───────────────────────────────────────────────────────────────────────────
Bertanggung jawab untuk semua operasi database.

Metode penting:

1. init_database()
   → Membuat tabel jika belum ada (otomatis saat start)
   → Dijalankan 1x saat aplikasi pertama kali dibuka

2. tambah_alat(nama_alat, stok, deskripsi)
   → Menambah alat baru
   → Validasi: nama tidak boleh duplikat
   → Return: (success, message)

3. tambah_peminjaman(nama_peminjam, id_alat, jumlah)
   → Menambah record peminjaman
   → Validasi: stok harus cukup
   → Otomatis kurangi stok alat
   → Return: (success, message)

4. kembalikan_alat(id_peminjaman)
   → Mengupdate status peminjaman menjadi "Dikembalikan"
   → Otomatis tambah stok alat
   → Return: (success, message)

5. get_all_alat(), get_all_peminjaman()
   → Fetch data dari database untuk ditampilkan di UI


B. CLASS SIJAtoolsApp
───────────────────────────────────────────────────────────────────────────
Bertanggung jawab untuk UI dan interaksi user.

Metode penting:

1. create_widgets()
   → Setup tab utama (4 tab)
   → Dipanggil 1x saat inisialisasi

2. create_tab_alat(), create_tab_peminjaman(), dll
   → Membuat UI untuk setiap tab
   → Setup event listener (button click, double click)

3. tambah_alat(), pinjam_alat(), kembalikan_alat()
   → Event handler untuk tombol
   → Validasi input dari user
   → Call database manager
   → Update UI (refresh tabel)

4. load_data_alat(), load_riwayat()
   → Refresh tabel dari database
   → Dipanggil setelah ada perubahan data


C. ALUR PEMINJAMAN ALAT
───────────────────────────────────────────────────────────────────────────

1. User login Tab "Input Peminjaman"
2. Isi form:
   - Nama peminjam
   - Pilih alat (dropdown)
   - Jumlah
3. Klik tombol "Pinjam"
4. Aplikasi:
   - Validasi stok
   - INSERT ke tabel peminjaman (status='Dipinjam')
   - UPDATE tabel alat (kurangi stok)
   - Refresh tabel
5. User melihat hasil di "Peminjaman Aktif"

PENGEMBALIAN:
1. User login Tab "Pengembalian Alat"
2. Pilih baris peminjaman
3. Klik "Kembalikan Alat"
4. Aplikasi:
   - UPDATE tabel peminjaman (status='Dikembalikan')
   - UPDATE tabel alat (tambah stok)
   - Refresh tabel
5. User melihat status berubah di "Riwayat Peminjaman"


D. VALIDASI & ERROR HANDLING
───────────────────────────────────────────────────────────────────────────

Input Validasi:
✓ Nama alat tidak boleh kosong
✓ Nama alat tidak boleh duplikat
✓ Stok harus angka positif
✓ Nama peminjam tidak boleh kosong
✓ Jumlah tidak boleh kosong & harus positif
✓ Stok tidak boleh minus (sistem cegah over-borrow)

Error Handling:
✓ Try-except pada operasi database
✓ Messagebox info/error untuk user
✓ Konfirmasi sebelum hapus/kembalikan

10. TROUBLESHOOTING
================================================================================

Problem 1: "ModuleNotFoundError: No module named 'tkinter'"
Solution: 
- Windows: pip install tk
- Linux: sudo apt-get install python3-tk
- macOS: Biasanya included, atau gunakan brew

Problem 2: Database kosong setelah buka aplikasi
Solution: Normal. Database auto-created kosong. Tambah alat di Tab 1.

Problem 3: Error "database is locked"
Solution: Tutup aplikasi, tunggu 5 detik, buka lagi. Jarang terjadi.

Problem 4: Text terpotong di layar kecil
Solution: Window bisa di-resize. Drag corner untuk membesar.
